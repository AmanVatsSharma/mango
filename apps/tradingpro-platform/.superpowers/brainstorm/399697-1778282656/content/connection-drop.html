WebSocket drops — what does the user see?

Feed disconnects happen. Prices go stale. The user might place an order on a 2-minute-old price. How do we handle it?

A

Sticky Banner — Non-blocking

A slim amber bar slides in at the top of the watchlist: "Feed paused — reconnecting…". Prices dim slightly. When reconnected, bar slides out with a brief "Live ✓" flash. Trading still allowed.

B

Block Market Orders — Allow Limit

Banner like A, but the order screen disables "Market" order type when feed is stale (>30s). Limit orders still allowed since the user sets the price. Protects against trading on bad data.

C

Smart Recovery — Auto-retry + REST Fallback

Banner like A. But behind the scenes: auto-reconnects every 3s, and if WS is down, silently fetches prices via REST every 15s so the watchlist never goes fully stale. User barely notices.
