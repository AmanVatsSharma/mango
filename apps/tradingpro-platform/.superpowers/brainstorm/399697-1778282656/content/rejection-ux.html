When an order gets rejected, what should happen?

Insufficient margin, market closed, exchange rejection — currently just a red toast for 7s. Which feels right?

A

Stay on Order Screen — Inline Error

Order screen stays open. Error banner slides in under the header with the exact reason. User can fix the issue (adjust quantity, switch to limit) and retry without reopening the drawer.

B

Red Bottom Card — Same as Success

Closes the order screen (like success), but the persistent bottom card turns red with the rejection reason. Consistent with the B choice above — one card handles all outcomes.

C

Both — Inline for Fixable Errors, Card for Others

Smart routing: if the user can fix it right there (reduce qty, switch order type) → stay on screen with inline error + smart suggestion. If it's a hard rejection (market closed, risk limit) → red card on watchlist.

Logic
Fixable → stay on screen, smart suggestion
Hard reject → red bottom card
Fixable errors: margin, price, qty
Hard errors: market closed, risk limit, segment disabled
